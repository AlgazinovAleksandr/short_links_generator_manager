# OMG, this is my yet another pet-project

The UI that generates a short version of the link. In addition, it provides cool features such as QR-code generation, and funny link customisation. Make sure to check self_promotion, and secret endpoints!

## Функционал сервиса

### Обязательные функции

#### 1. Фкнкционал коротких ссылок

| Метод | Эндпоинт | Описание |

| POST | /links/shorten | Создать короткую ссылку |

| PUT | /links/{short_code} | Обновить оригинальный URL или переименовать код ссылки |

| DELETE | /links/{short_code} | Удалить ссылку |

Изменение и удаление доступны только авторизованному владельцу ссылки

#### 2. Статистика по ссылке

GET /links/{short_code}/stats

Возвращает оригинальный URL, дату создания, количество переходов (click_count) и дату последнего перехода (last_accessed_at).

#### 3. Кастомный алиас

POST /links/shorten с полем custom_alias

Позволяет задать произвольный короткий код вместо автоматически сгенерированного. Проверяется уникальность алиаса.

#### 4. Поиск по оригинальному URL

GET /links/search?original_url={url}

Находит короткую ссылку по полному оригинальному URL. Если ссылка истекла, возвращает 404.

### Дополнительные функции (на бонусные баллы, всем же хочется иметь высокую оценку за курс :))

#### Favorite Word — любимое слово в коде ссылки

| Метод | Эндпоинт | Описание |

| POST | /auth/me/favorite-word | Установить любимое слово |

| DELETE | /auth/me/favorite-word | Сбросить любимое слово |

Если у авторизованного пользователя задано любимое слово, оно автоматически добавляется в начало каждого нового сгенерированного short_code. Например, если слово SPONGEBOB, то код ссылки будет SPONGEBOBabc123 (например, ведь оставшаяся часть кода генерируется автоматически). На кастомные алиасы не влияет.

#### QR-код для ссылки

GET /links/{short_code}/qr

Генерирует QR-код для указанной короткой ссылки и возвращает PNG-изображение.

#### Случайный анекдот

GET /extras/joke

Возвращает случайный анекдот из файла jokes.md. Анекдоты разделены разделителем ILOVEPIZZA (ведь мы этого достойны, ведь я люблю пиццу, но только не папа джонс там теста больше чем начинки это кошмар).

#### Self Promotion

GET /extras/self-promotion

Возвращает ссылку на мой канал) Самая важная функция в проекте!

#### Secret URL

GET /extras/secret-url

Секретный URL. Попробуй сам — что же там? Подсказка: ну конечно же это не Рик-Рол never gonna give you up never gonna let you down

## Регистрация и аутентификация

| Метод | Эндпоинт | Описание |

| POST | /auth/register | Зарегистрировать пользователя |

| POST | /auth/login | Войти и получить JWT-токен |

| GET | /auth/me | Получить профиль текущего пользователя |

Аутентификация реализована через JWT (Bearer-токен). Токен передаётся в заголовке Authorization: Bearer <token>.

## База данных

В качестве БД используется PostgreSQL. Работаем с ней через SQLAlchemy в асинхронном режиме (async engine + AsyncSession). Сессия живет в src/database.py. Миграции осуществляются через Alembic (папка alembic/), единственная миграция — 001_initial_schema. Кстати очень удобная штука, до этого не пользовался и стала приятным открытием!)

Две таблицы:

**users**

| Колонка | Тип | Описание |

| id | UUID | Первичный ключ |

| username | String(50) | Уникальный логин, индексирован |

| email | String(255) | Уникальный email |

| hashed_password | String(255) | Хэш пароля (не храним пароли в открытом виде, мы же не варвары в конце то концов) |

| favorite_word | String(100) | Любимое слово (например, TIKKAMASALAISGOOD) для префикса short_code, опционально |

| created_at | DateTime | Дата регистрации |

**links**

| Колонка | Тип | Описание |

| id | UUID | Первичный ключ |

| original_url | String(2048) | Оригинальный URL |

| short_code | String(50) | Короткий код, уникальный, индексирован |

| user_id | UUID (FK) | Владелец ссылки, опционально (анонимные ссылки тоже живут) |

| created_at | DateTime | Дата создания |

| expires_at | DateTime | Дата истечения, опционально |

| click_count | Integer | Счетчик переходов |

| last_accessed_at | DateTime | Дата последнего перехода |

Помимо PostgreSQL, Redis используется как кэш для хранения маппинга short_code в original_url, чтобы не ходить в БД при каждом редиректе (хотя бы где-то все будет красиво)

**Стэк: Python, FastAPI, PostgreSQL, SQLAlchemy (async), Redis, Alembic, Docker, JWT, Pydantic**

## Тесты

**Здесь написана лишь краткая выжимка, сильно сильно подробнее и вообще со всеми возможными и невозможными деталями это можно почитать в tests/README_testing.md**

Все связанное с тестами есть в папке tests. По итогам замером, функционал покрыт тестами на **98%** (измеряется по папке src/). Тесты разделены на три категории: юнит-тесты (проверяют отдельные функции в изоляции), функциональные тесты (проверяют API от HTTP-запроса до ответа через тестовый клиент) и нагрузочные тесты на базе Locust. Тестовая база данных — SQLite в памяти, Redis заменён на in-memory мок. Запустить все тесты и получить отчёт о покрытии одной командой:

pytest tests/

При запуске этой команды образуется папка htmlcov, где куча всякого мусора. Поэтому она отправляется прямиком в gitignore, а все нужные файлы (по отчетам о тестировании) были перенесены в отдельную папку coverage_report

HTML-отчет о покрытии (сводная таблица) доступен в папке coverage_report/index.html без запуска тестов

Посмотреть отчет: open coverage_report/index.html

Подробнее — в tests/README_testing.md
