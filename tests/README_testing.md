# Тесты для нашего прекрасного и замечательного и прям не могу какого сервиса коротких ссылок

### conftest.py

- поднимает тестовую базу данных SQLite в памяти вместо PostgreSQL

- заменяет Redis на простой in-memory мок (mock_redis), чтобы тесты не зависели от внешних сервисов

- создает HTTP-клиент для обращения к приложению в тестовом режиме

- предоставляет вспомогательные штуки: зарегистрированный пользователь, токен авторизации, заголовки авторизации

### functional/ — функциональные тесты

Тесты этой папки проверяют работу API целиком: от HTTP-запроса до ответа. Они имитируют реальное поведение пользователя, обращаясь к эндпоинтам через тестовый клиент.

#### test_auth.py

Проверяет всё, что связано с аутентификацией:

- регистрация нового пользователя (нормальная ситуация / дубликаты)

- вход в систему (нормальная ситуация / неверный пароль / несуществующий пользователь / ВСЕ СЛОМАЛОСЬ)

- получение данных текущего пользователя

- установка и сброс любимого слова пользователя

#### test_links.py

Проверяет весь жизненный цикл короткой ссылки:

- создание ссылки (анонимно / с кастомным алиасом / со сроком годности / и так далее)

- получение статистики по ссылке

- поиск ссылки по оригинальному URL

- генерация QR-кода

- использование любимого слова при создании ссылки

- и много много много всего другого

#### test_extras.py

Проверяет дополнительные эндпоинты:

- получение случайного анекдота (GET /extras/joke)

- эндпоинт самопиара (ооо даа) (GET /extras/self-promotion)

- секретный URL (это ни в коем случае не Рик-Рол!) (GET /extras/secret-url)

- корневой эндпоинт (GET /)

### unit/ — юнит-тесты

Тесты этой папки проверяют отдельные функции и роутеры в изоляции от HTTP-слоя (потому что иначе покрытие функционала будет низким).

#### test_short_code.py

Проверяет функции из src/services/link_service.py:

- длина сгенерированного короткого кода (должна быть 8 символов)

- допустимые символы в коде (только буквы и цифры)

- уникальность: из 1000 сгенерированных кодов менее 1% совпадений

- функция apply_favorite_word — корректно вставляет слово в домен URL (для https и http)

- функция is_link_expired — корректно определяет, истекла ли ссылка (нет даты, в будущем, в прошлом)

#### test_routes_direct.py

Вызывает функции роутеров напрямую, минуя HTTP

Тесты покрывают:

- **Регистрация**: обычная ситуация, дубликат username, дубликат email

- **Вход**: клassическая ситуация, неверный пароль

- **Любимое слово**: установка и сброс

- **Создание ссылки**: базовое, с кастомным алиасом, конфликт алиасов

- **Поиск по URL**: найдено, не найдено, истекшая ссылка

- **Статистика**: найдено, не найдено, истекшая ссылка

- **QR-код**: найдено, не найдено, истекшая ссылка

- **Обновление ссылки**: смена URL, смена кода, конфликт кода, не найдено, чужая ссылка

- **Удаление ссылки**: успех, не найдено, чужая ссылка

#### test_joke_service.py
Проверяет функции из src/services/joke_service.py:

- функция get_random_joke возвращает строку из файла с анекдотами (замечательная вещь)

- при пустом файле возвращается строка-заглушка

- константа-разделитель это "ILOVEPIZZA", потому что тильды и всякие там точки с запятой это скучно очень

### `load/` — нагрузочные тесты

#### `locustfile.py`

Содержит сценарий нагрузочного тестирования на базе [Locust](https://locust.io/). Каждый виртуальный пользователь:

- Регистрируется и логинится при старте

- Случайным образом выполняет задачи с разными весами:

- создание ссылки

- редирект по короткому коду

- получение статистики

- поиск по URL

- получение анекдота (самое важное, но вес все же я поставил минимальный для этого типа развлечений)

- создание ссылки с кастомным алиасом

Нагрузочные тесты запускаются отдельно и требуют работающего сервера

В pytest.ini прописан addopts = --cov=src --cov-report=html, поэтому покрытие измеряется автоматически при каждом запуске pytest Строки, помеченные # pragma: no cover, исключаются из покрытия тестами, так как многие вещи либо написались но по итогу не использовались, либо изначально писались просто для развлечения (а то без этого было бы прям совсем-совсем грустно). Например, функция, которая делает примпт смешной юморески, ну или там генератор случайного числа)

## Как запустить тесты

pip install -r requirements.txt

pytest tests/

### Покрытие кода

Покрытие измеряется автоматически при запуске pytest (настроено через addopts в pytest.ini). HTML-отчёт сохраняется в папку htmlcov/:

open htmlcov/index.html - увидеть результаты прогона тестов, узнать покрытие

Так как в htmlcov куча мусора и ненужных файлов, эта папка лежит в gitignore, а результаты отчета можно посмотреть в отдельно созданной мной папке coverage_report/:

open coverage_report/index.html

Текущий результат: **98% покрытия** кода в src/
